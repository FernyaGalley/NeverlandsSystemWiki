*Tenses (v): resolved (past), resolve (present), resolving (progressive)*
*Laconic: Figuring out if something is Canon.*

To **resolve** a [[Statements|Statement]] is to decide how a Statement is [[to canonize|canonized]] or [[to reject|rejected]] from [[Canons|Canon]]. During resolution, the [[Tables|Table]] and the [[Directors]] decide whether or not the top Statement of the [[Racks|Rack]] makes sense, given the current state of Canon. *In other words,* the Directors must resolve [[Arbitration]].

While Statements are resolving, the Directors should [[to close|close]] the [[Conflict State Handbook|Conflict State]]. If the Statement has no [[to challenge|challenges]] and seems logically [[to validate|valid]], then it should be [[to accept|accepted]] into Canon. If the Directors feel like there might be some caveats, they'll raise concerns in this step, [[to melt|melting]] or [[to wick|wicking]] a Statement as necessary to get [[to clarify|clarifying]] information. If the Director thinks a challenge is credible, they should reject the Statement; if appropriate, they should return that [[Props|Prop]] to the [[Appropriateness and Relevance|appropriate]] [[Pockets|Pocket]].

Whatever the [[Results|Result]] of Arbitration, when the Directors are done resolving a Statement they should [[to open|open]] the Conflict State once more.
## Feature - (Resolved)
*see also*

When a [[Statements|Statement]] has been resolved, it is [[to imply|implied]] to have (Resolved) [[to introduce|introduced]] as a [[Features|Feature]]. This is a self-documenting Feature, as when a Statement is [[to canonize|canonized]] it will be in [[Canons|Canon]]. Any other [[Results|Result]] will see the Statement leave the [[Racks|Rack]] and potentially return to a [[Pockets|Pocket]].
